Pixel-Directional Animator (PDAnimator)

PDAnimator is a concoction of a custom Unity Animator that makes it simpler to use 2D-topdown-spritesheet animation (Think the original Legend of Zelda), along with some tooling for convenience.
This project contains 3 separate units from a personal project:

  • A custom Animator that replaces Unity's default one (PDAnimator)
  • A tool that speeds up setup work for creating sprite sheets (Spritesheet Slicer)
  • A tool that automates creating animations from a sprite sheet (PDA Animation Wizard)

PDAnimator

PDAnimator replaces Unity's default animator with a version that is more friendly to sprite use. It handles 4-directional animation states in a state machine.
Instead of manually creating animations, slicing them individually, setting their pivots one by one, and dragging in every single frame in the Unity animation editor, create animation clips and insert the frames. Then make states that use these clips using the pre-built classes.

This system has proven so useful in this project that I have re-written it to Godot Engine for a future project due to similar frustrations; it taught me how to design editor tools, along with teaching me how to design easily understandable library code that is meant to be extended.


Tools

The Spritesheet slicer is an editor tool that automates setting slicing metadata on Unity sprite assets. This is something that becomes incredibly tedious, due to the auto-slicing in Unity being deeply flawed (at least in the Unity version this was developed for).

The Animation Wizard is used to very easily create animation files for PDAnimator in a drag-and-drop fashion. It automates creating Scriptable Objects and setting their properties in bulk, instead of manually setting them one-by-one.
